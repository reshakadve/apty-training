Functions and Error Handling

Topics

In JavaScript, a function is a block of code designed to perform a particular task. Functions are one of the fundamental building blocks in JavaScript programming and are used to structure code logically.

1. Function Declaration, Expression, and Arrow Functions

Function Declaration

A function declaration defines a function that can be called later in the program. It is declared using the function keyword. 

function greet(name) {
    console.log("Hello, " + name + "!");
}

greet('Alice');

In the above example, greet is the function name, and name is the parameter. The function logs a greeting message to the console when called.

Function Expression

A function expression involves creating a function as part of an expression, often assigned to a variable. 

const greet = function(name) {
    console.log("Hello, " + name + "!");
};

greet('Bob');

Here, greet is a function expression stored in a variable. This function can be invoked using the variable name.

Arrow Functions

Arrow functions are a shorter syntax for writing functions. They are especially useful for anonymous functions and maintaining the correct this context. 

const greet = (name) => {
    console.log("Hello, " + name + "!");
};

greet('Charlie');

In the example above, the arrow function has the same functionality as the previous examples but with a more concise syntax.

2. Parameters, Arguments, and Default Values

Function Parameters and Arguments

Parameters are placeholders in a function definition, and arguments are the actual values passed to the function when called. 

function greet(name) {
    console.log("Hello, " + name + "!");
}

greet('David');  // 'David' is the argument

Here, name is the parameter, and 'David' is the argument passed to the function.

Default Function Parameters

JavaScript allows setting default values for parameters. If an argument is not provided for a parameter, the default value will be used. 

function greet(name = 'Guest') {
    console.log("Hello, " + name + "!");
}

greet();        // Uses default value 'Guest'
greet('Emily'); // Uses provided argument 'Emily'

In the example, if no argument is passed, the function will use the default value of 'Guest'.

3. Closures and Lexical Scoping

Closures

A closure is a function that remembers its lexical scope, even when the function is executed outside that scope. This is useful for data encapsulation and maintaining state across function calls. 

function outer() {
    let counter = 0;

    return function inner() {
        counter++;
        console.log(counter);
    };
}

const increment = outer();  // Creates a closure
increment(); // Outputs: 1
increment(); // Outputs: 2

In this example, inner is a closure because it "remembers" the counter variable from its lexical scope, even though it's executed outside of the outer function.

Lexical Scoping

Lexical scoping means that variables are resolved based on their location within the source code. Inner functions have access to variables defined in their outer (enclosing) scope. 

function outer() {
    let outerVar = 'I am outside!';

    function inner() {
        console.log(outerVar);  // Accesses the outer variable
    }

    inner();
}

outer(); // Outputs: 'I am outside!'

In this example, the inner function can access the variable outerVar because of lexical scoping.

4. Intermediate Topics

Higher-order Functions

A higher-order function is a function that either takes one or more functions as arguments or returns a function as its result. 

function add(a, b) {
    return a + b;
}

function calculate(operation, x, y) {
    return operation(x, y);
}

console.log(calculate(add, 5, 3));  // Outputs: 8

In the example, calculate is a higher-order function because it takes the add function as an argument.

Function Expressions vs Function Declarations

Function expressions are functions that are created and assigned to a variable, while function declarations are named functions defined using the function keyword. 

console.log(a()); // Works because of hoisting

function a() {
    return 'Hello from function declaration!';
}

console.log(b()); // Throws error: b is not a function

var b = function() {
    return 'Hello from function expression!';
};

Function declarations are hoisted to the top of their scope, while function expressions are not. In the example, the declaration of a is hoisted, but b is not.

5. Error Handling

Syntax Errors

Syntax errors occur when the JavaScript code is written incorrectly, violating the syntax rules of the language.

Example:

var a = 10 ++ 10 // Uncaught SyntaxError: Invalid left-hand side expression in postfix operation

How to Handle: Fix the syntax error by correcting the code.

Runtime Errors

Runtime errors occur when the script is syntactically correct but fails during execution due to unexpected behavior.

Example:

a += 10 // Uncaught ReferenceError: a is not defined let a = 10

How to Handle: Use a try...catch block to handle runtime errors gracefully.

Logical Errors

Logical errors occur when the code runs without issues but produces incorrect or unintended results.

Example:

let a = 10;
let b = 20;
let sum = a - b; // Logical error: subtraction instead of addition

How to Handle: Debug the code and use tools like breakpoints or console logs to identify the logic issue.

try...catch Statement

The try...catch statement allows you to test a block of code for errors and handle them without crashing the program.

Example:

try {
  let a = 10;
  console.log(b); // ReferenceError
} catch (error) {
  console.log("An error occurred:", error.message);
}
finally Block

The finally block executes after the try and catch blocks, regardless of whether an error occurred.

Example:

try {
  let a = 10;
  console.log(b); // ReferenceError
} catch (error) {
  console.log("An error occurred:", error.message);
} finally {
  console.log("Execution completed.");
}
Throwing Custom Errors

You can use the throw statement to create and throw custom errors.

Example:

function checkAge(age) {
  if (age < 18) {
    throw new Error("Age must be 18 or older.");
  }
}

try {
  checkAge(16);
} catch (error) {
  console.log(error.message);
}
Error Objects

Error objects provide information about errors and can be used in the catch block.

Example:

try {
  let a = 10;
  console.log(b); // ReferenceError
} catch (error) {
  console.log("Name:", error.name); // ReferenceError
  console.log("Message:", error.message);
}
Custom Error Classes

Custom error classes allow you to create error types specific to your application.

Example:

class ValidationError extends Error {
  constructor(message) {
    super(message);
    this.name = "ValidationError";
  }
}

try {
  throw new ValidationError("Invalid input!");
} catch (error) {
  console.log(error.name);
  console.log(error.message);
}
