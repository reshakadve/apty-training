Topics

In HTML, a markup language is used to structure content on the web. These tags and elements form the backbone of web development, allowing developers to define headings, paragraphs, lists, tables, forms, and more with precision.

1. Structure of an HTML Document
  • Overview: An HTML document is structured using a standard template that includes a declaration, head, and body sections. This structure ensures proper rendering by web browsers.
  • Key Elements:
    • <!DOCTYPE html>: Declares the document as HTML5. No attributes.
    • <html>: Root element of the HTML page.
      • Attributes:
        • lang: Specifies the language (e.g., "en", "es", "fr").
    • <head>: Contains metadata and document information.
      • Attributes: None directly, but contains elements like <meta>, <title>, etc.
    • <body>: Contains the visible content (text, images, links, etc.).
      • Attributes:
        • bgcolor: Sets background color (e.g., "#ffffff", "red"). Deprecated in HTML5, use CSS instead.
        • background: Sets background image URL. Deprecated in HTML5, use CSS instead.
  • Explanation: The <!DOCTYPE html> declaration ensures compatibility with HTML5 standards. The <html> tag wraps the entire document, with the lang attribute aiding accessibility and SEO. The <head> section includes metadata (e.g., character encoding via <meta charset="UTF-8">) and the document title (<title>). The <body> holds all visible content, with legacy attributes like bgcolor now replaced by CSS for modern development.
2. Tags, Elements, Attributes
  • Basic Tags:
    • <html>: See above structure.
    • <head>: See above structure.
    • <body>: See above structure.
    • <title>: Defines the document title.
      • Attributes: None.
    • <h1> to <h6>: Heading levels 1 to 6 (h1 being the largest).
      • Attributes:
        • align: Alignment (e.g., "left", "center", "right", "justify"). Deprecated in HTML5, use CSS.
    • <p>: Paragraph tag.
      • Attributes:
        • align: Alignment (e.g., "left", "center", "right", "justify"). Deprecated in HTML5, use CSS.
    • <!-- -->: Comment tag. No attributes.
  • Explanation: Tags are the building blocks of HTML, enclosed in angle brackets (e.g., <p>). Elements include the opening tag, content, and closing tag (e.g., <p>Text</p>). Attributes provide additional information (e.g., align in older HTML) and are specified within the opening tag. Modern HTML5 favors CSS for styling over deprecated attributes like align.
3. Ordered and Unordered Lists (<ol>, <ul>, <li>)
  • <ol>: Ordered list.
    • Attributes:
      • type: List style (e.g., "1", "A", "a", "I", "i").
      • start: Starting number (e.g., "1", "5").
      • reversed: Reverses order (boolean: no value, just presence).
  • <ul>: Unordered list.
    • Attributes:
      • type: Bullet style (e.g., "disc", "circle", "square"). Deprecated in HTML5, use CSS.
  • <li>: List item.
    • Attributes:
      • value: Sets the value of the list item (numeric, for <ol>).
  • Explanation: <ol> creates numbered lists, with type and start customizing the numbering style and starting point. <ul> creates bullet-point lists, with type (now deprecated) controlling bullet styles. <li> defines each item, and value overrides the default numbering sequence. Nesting lists is common for hierarchical data.
4. Nesting Lists
  • Overview: Nesting involves placing one list inside another to create hierarchical structures.
  • Example: 
                <ul>
                <li>Item 1
                    <ul>
                        <li>Subitem 1.1</li>
                        <li>Subitem 1.2</li>
                    </ul>
                </li>
                <li>Item 2</li>
            </ul>
  • Explanation: Nesting allows for complex, organized content (e.g., outlines or menus). Use <ul> or <ol> inside <li> tags. Ensure proper closing of tags to avoid rendering issues. CSS can style nested lists differently for visual hierarchy.
5. Adding Images (<img>) and Alt Text
  • <img>: Embeds an image.
    • Attributes:
      • src: Image URL (e.g., "image.jpg", "https://example.com/image.png").
      • alt: Alternative text for accessibility (e.g., "A beautiful landscape").
      • width: Image width in pixels or percentage (e.g., "300", "50%").
      • height: Image height in pixels or percentage (e.g., "200", "50%").
      • title: Tooltip text (e.g., "Hover text").
      • class: CSS class name (e.g., "image-style").
      • id: Unique identifier (e.g., "main-image").
  • Explanation: The <img> tag is self-closing and requires a src attribute to specify the image source. The alt attribute is crucial for accessibility, describing the image for screen readers. width and height control size, while title provides hover text. Use CSS for advanced styling.
6. Creating Tables (<table>, <tr>, <td>, <th>, <caption>)
  • <table>: Defines a table.
    • Attributes:
      • border: Border thickness (e.g., "1", "0"). Deprecated in HTML5, use CSS.
      • width: Table width (e.g., "100%", "500px"). Deprecated in HTML5, use CSS.
  • <tr>: Table row. No unique attributes.
  • <th>: Header cell.
    • Attributes:
      • colspan: Number of columns to span (e.g., "2").
      • rowspan: Number of rows to span (e.g., "2").
      • scope: Defines the cell's scope (e.g., "col", "row", "colgroup", "rowgroup").
  • <td>: Data cell. Same attributes as <th> (e.g., colspan, rowspan).
  • <caption>: Table caption.
    • Attributes: None.
  • Explanation: Tables organize data in rows (<tr>) and columns using <td> for data and <th> for headers. The <caption> provides a title. Attributes like colspan and rowspan merge cells, while scope improves accessibility. Modern styling uses CSS instead of deprecated attributes.
7. Form Basics: <form>, <input>, <label>, <button>
  • <form>: Creates a form for user input.
    • Attributes:
      • action: URL to submit data (e.g., "/submit").
      • method: HTTP method (e.g., "get", "post").
      • enctype: Encoding type (e.g., "application/x-www-form-urlencoded", "multipart/form-data" for files).
  • <input>: Input field.
    • Attributes:
      • type: Input type (e.g., "text", "password", "checkbox", "radio", "submit", "file", "email", "number", "date").
      • name: Field name (e.g., "username").
      • value: Default value (e.g., "Enter text").
      • placeholder: Hint text (e.g., "Type here").
      • required: Makes field mandatory (boolean).
      • disabled: Disables input (boolean).
  • <label>: Label for an input.
    • Attributes:
      • for: Associates with input ID (e.g., "username").
  • <button>: Submit or reset button.
    • Attributes:
      • type: Button type (e.g., "submit", "reset", "button").
      • value: Button value (e.g., "Submit").
      • disabled: Disables button (boolean).
  • Explanation: <form> collects user data, with action and method defining submission behavior. <input> supports various types for different inputs, with attributes like required ensuring validation. <label> improves accessibility by linking to inputs via for. <button> triggers actions, with type specifying its role.
