Using <iframe> to Embed Content
  • Overview: The src attribute specifies the embedded content’s URL, width and height define dimensions, and loading improves performance by deferring iframe loading.

  • Explanation:
    • Specify Content with src:
      • Defines the source URL of the embedded page.
      • Supports embedding websites, videos, maps, or documents.
      • Example: <iframe src="https://example.com"></iframe>
    • Adjust Size with width and height:
      • Defines the iframe's display size in pixels or percentages.
      • Responsive design benefits from percentage-based dimensions.
      • Example: <iframe src="https://example.com" width="600" height="400"></iframe>
    • Improve Performance with loading:
      • Delays loading until needed (lazy) or loads immediately (eager).
      • Enhances page speed by deferring non-essential iframes.
      • Example: <iframe src="https://example.com" loading="lazy"></iframe>
Attributes: sandbox, allow, srcdoc
  • Overview: The sandbox attribute restricts iframe actions for security, allow grants specific permissions (e.g., camera, fullscreen), and srcdoc lets you embed inline HTML instead of an external source.

  • Explanation:
    • Enable sandbox for Security:
      • Restricts embedded content from running scripts or accessing sensitive data.
      • Example: <iframe src="https://example.com" sandbox="allow-scripts allow-same-origin"></iframe>
    • Restrict Permissions with allow:
      • Controls features the embedded content can access.
        • geolocation – Allows the iframe to access the user’s location (useful for maps and delivery apps).
        • camera – Enables video input, essential for video calls and AR applications.
        • microphone – Allows audio input, required for voice recording and online meetings.
        • fullscreen – Enables the iframe content to enter fullscreen mode (useful for videos and games).
        • encrypted-media – Required for playing DRM-protected media (Netflix, YouTube, etc.).
        • autoplay – Allows media (audio/video) to play automatically without user interaction.
        • payment – Enables integration with browser-based payment APIs (e-commerce transactions).
        • publickey-credentials-get – Needed for passwordless authentication (WebAuthn, security keys).
        • usb – Allows communication with USB devices (e.g., POS systems, hardware wallets).
        • screen-wake-lock – Prevents the screen from dimming or locking (ideal for video calls and presentations).
      • Example: <iframe src="https://example.com" allow="fullscreen"></iframe>
    • Use srcdoc for Inline HTML Content:
      • Allows embedding HTML directly inside an iframe without requiring an external source.
      • Useful for rendering small, self-contained HTML snippets.
      • Example: <iframe srcdoc="<h1>Hello, World!</h1><p>This is embedded content.</p>" width="400" height="200"></iframe>
Shadow DOM: encapsulation and styling
  • Overview: The Shadow DOM enables encapsulation of HTML, CSS, and JavaScript, preventing styles and scripts from leaking or interfering with the main document.

  • Explanation:
    • Encapsulate Components with Shadow DOM:
      • Creates an isolated DOM tree within an element.
      • Keeps styles and scripts scoped to the component.
      • Example: const shadowHost =document.querySelector("#shadow-host");
        const shadowRoot= shadowHost.attachShadow({ mode: "open" });
        shadowRoot.innerHTML = `Inside Shadow DOM`;
    • Scoped Styling in Shadow DOM:
      • Styles defined inside the Shadow DOM do not affect the main document.
      • Prevents conflicts with global styles.
      • Example: shadowRoot.innerHTML = `Styled inside Shadow DOM`;
    • Use mode: "open" or mode: "closed":
      • open: Allows JavaScript to access the Shadow DOM.
      • closed: Prevents external access to the Shadow DOM.
      • Example: const closedShadow = element.attachShadow({ mode: "closed" }); console.log(element.shadowRoot); // null
Using <template> for Reusable Content
  • Overview: The <template> element is used to define reusable chunks of HTML that are not rendered until they are explicitly used via JavaScript.
    • Content inside a template is not rendered until it is cloned and added to the DOM.
    • Helps improve performance by reducing unnecessary DOM elements at page load.
    • Commonly used for dynamically inserting UI components, such as cards, modals, or lists.
  • Explanation:
    • Define Reusable HTML Structures:
      • Use template for elements that need to be inserted multiple times dynamically.
    • Use JavaScript to Clone and Append Content:
      • Content inside template is inactive and must be cloned before adding to the DOM.
      • Example:
        <template id="user-card">
        <div class="card">
        <h2>User Name</h2>
        <p>User details go here.</p>
        </div>
        </template>
        <script>
        document.addEventListener("DOMContentLoaded", () => {
        const template = document.getElementById('user-card');
        const clone = template.content.cloneNode(true);
        document.body.appendChild(clone);
        });
        </script>
    • Ensure the Template is Hidden by Default:
      • Unlike div or span, the content inside template does not render immediately.
    • Use Templates for Dynamic Lists:
      • Useful for generating multiple elements, such as dynamically created lists or UI components.
